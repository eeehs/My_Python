- 파일 현재 상태를 기록해두려면
    git add 파일명
    git commit -m '메모'
- branch 생성
    git branch (이름)
- branch switch
    git switch (이름)
- branch merge
    기준이 되는 branch로 이동 후, git merge (브랜치명)
- 현재 branch의 이름 변경
    git branch -M (변경할 이름)

- 충돌 해결은 코드고치고 git add & git commit

- rebase 쓰는 이유
    - 3-way merge는 나중에 git log를 출력할 때 복잡해보임
    - 간단하고 짧은 브랜치들은 rebase를 사용하면 깔끔해보임

- rebase 단점
    - conflict가 많이 발생함. 

- rebase & merge 방법
    1. 새로운 브랜치로 이동
    2. git rebase (중심브랜치명)
    3. 중심 브랜치로 이동
    4. git merge (새로운 브랜치명)

- 과거로 모든걸 되돌리기
    git reset --hard 커밋아이디

- 원격저장소 push
    git push -u 원격저장소주소 올릴로컬브랜치명
